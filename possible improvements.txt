Possible Improvements for CirnOS
Review date: 2026-02-15

1) High-priority security and safety
- Harden SSH in `modules/common.nix`: `services.openssh.settings.PasswordAuthentication = true` is risky if host port 22 is reachable. Prefer SSH keys only (`PasswordAuthentication = false`) and keep `PermitRootLogin = "no"`.
- Consider host-specific SSH exposure: SSH is enabled globally in `modules/common.nix`. If the laptop does not need inbound SSH on public networks, disable SSH on that host.
- Move firewall openings to host scope: `modules/firewall.nix` opens TCP/UDP 7777 and TCP 22 for every machine. Better to open only where needed.
- Re-check insecure package exception in `modules/common.nix`: `qtwebengine-5.15.19` is explicitly permitted for "teamspeak3", but the package list now uses `teamspeak6-client` in `modules/programs.nix`. If no package still needs it, remove the insecure exception.
- Auto-upgrade path hardening in `modules/common.nix`: `system.autoUpgrade.flake = "/home/krieger/CirnOS#..."` runs as root from a user-writable path. Safer pattern is a root-owned checkout/symlink (for example under `/etc/nixos`).

2) Reliability and maintainability
- Remove duplicated package declarations in `modules/programs.nix`:
  - `programs.chromium.enable = true` plus `chromium` in `environment.systemPackages`.
  - `programs.partition-manager.enable = true` plus `kdePackages.partitionmanager` in `environment.systemPackages`.
- Clean up legacy channel setting in `modules/common.nix`: `nix.nixPath = [ "nixpkgs=/etc/channels/nixpkgs" ];` is usually unnecessary in a flake-based setup.
- Add SSD maintenance (`services.fstrim.enable = true`) if both systems are on SSD/NVMe.
- Add boot entry retention limit (`boot.loader.systemd-boot.configurationLimit`) to prevent systemd-boot menu growth.
- Both hosts currently have `swapDevices = [ ]` in their hardware configs. Consider `zramSwap.enable = true` (or a swap file/partition) for OOM resilience.

3) Host-specific correctness and behavior
- Split NVIDIA-specific Niri environment from `home/niri.nix` into desktop-only config:
  - `LIBVA_DRIVER_NAME=nvidia`, `GBM_BACKEND=nvidia-drm`, `__GLX_VENDOR_LIBRARY_NAME=nvidia`, and `WLR_NO_HARDWARE_CURSORS=1` are applied to all hosts, including the Intel laptop.
- Display layout in `home/niri.nix` is hardcoded to `DP-5/DP-4/DP-6`. This should ideally be host-specific (desktop vs laptop) to reduce drift and startup surprises.
- `DISPLAY=:0` in `home/niri.nix` can be brittle. Prefer not hardcoding unless you have a proven reason this is required.

4) Potential bug candidates
- In `home/noctalia/niri-focus-ring-live.nix`, `xdg.configFile."niri-config".enable = lib.mkForce false;` looks suspicious. If the intended target is Niri config management, it may need to match the actual path key (`niri/config.kdl`).
- `home/niri-wallpaper.nix` wallpaper service waits for workspace events and may miss setting a wallpaper immediately if no activation event appears at start. Consider applying one initial wallpaper before entering the event loop.
- In `home/default-apps.nix`, archive MIME defaults are set to Nautilus; GNOME file manager is fine, but an archive app integration (`file-roller`/`ark`) may provide better open/extract behavior for more formats.

5) Reproducibility and update strategy
- Shell aliases in `home/shellAliases.nix` use `--impure` for rebuild/update. If not strictly required, drop `--impure` to improve reproducibility.
- `home/comfyui.nix` uses runtime `git clone/pull` and `pip install --upgrade` (unpinned). This is convenient but non-reproducible and can break unexpectedly. Consider pinning repo/dependency versions or wrapping with a more reproducible workflow.
- In `home/comfyui.nix`, dependency bootstrapping marker file (`.deps-installed-v4`) is manual-versioned. It works, but consider a clearer upgrade path (for example pinned requirements lock + explicit update command).

6) Optional software/tools you may want
- Nix workflow tooling: `nh`, `nix-output-monitor`, `nvd`, `nix-tree`, `alejandra`, `statix`, `deadnix`.
- Security/ops additions (if relevant): `fail2ban` (when SSH is exposed), `fwupd` (especially on laptop), `smartmontools`, `lm_sensors`.
- Backup strategy package set: `restic` or `borgbackup` (+ remote target tooling) for automated backups.

7) Minor cleanup
- In `flake.nix`, comments mention NVIDIA beta vs kernel pin. Re-check regularly so pinning does not outlive the driver issue.
- In `modules/programs.nix`, both stable Discord and PTB/Canary are installed. If not intentionally needed together, removing extras reduces closure size and update churn.
